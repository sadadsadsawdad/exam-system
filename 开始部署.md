# 🚀 开始部署到 Railway + Vercel

## 📋 你需要准备的

1. **GitHub 账号**（免费）
2. **Railway 账号**（免费，用 GitHub 登录）
3. **Vercel 账号**（免费，用 GitHub 登录）
4. **10 分钟时间**

---

## 🎯 部署流程（3 步完成）

```
第 1 步：推送代码到 GitHub
    ↓
第 2 步：部署后端到 Railway（5分钟）
    ↓
第 3 步：部署前端到 Vercel（3分钟）
    ↓
完成！🎉
```

---

## 第 1 步：推送代码到 GitHub

### 如果你还没有 GitHub 仓库：

1. 访问 https://github.com/new
2. 创建新仓库（可以是私有的）
3. 在项目目录执行：

```bash
git init
git add .
git commit -m "准备部署"
git remote add origin https://github.com/你的用户名/你的仓库名.git
git branch -M main
git push -u origin main
```

### 如果已有仓库：

```bash
git add .
git commit -m "准备部署"
git push origin main
```

✅ 完成后，在 GitHub 上能看到你的代码

---

## 第 2 步：部署后端到 Railway

### 2.1 创建 Railway 项目

1. 访问 https://railway.app/
2. 点击 "Login" → "Login with GitHub"
3. 授权 Railway 访问你的 GitHub
4. 点击 "New Project"
5. 选择 "Deploy from GitHub repo"
6. 选择你的项目仓库
7. Railway 会自动开始构建（等待 3-5 分钟）

### 2.2 添加 MySQL 数据库

1. 在项目页面，点击右上角 "+ New"
2. 选择 "Database" → "Add MySQL"
3. 等待 MySQL 创建完成（约 1 分钟）

### 2.3 配置环境变量

1. 点击你的后端服务（springboot）
2. 点击 "Variables" 标签
3. 点击 "Raw Editor"
4. 粘贴以下内容：

```bash
SPRING_DATASOURCE_URL=jdbc:mysql://${{MySQL.MYSQL_HOST}}:${{MySQL.MYSQL_PORT}}/${{MySQL.MYSQL_DATABASE}}?useSSL=false&serverTimezone=Asia/Shanghai&characterEncoding=utf8&allowPublicKeyRetrieval=true
SPRING_DATASOURCE_USERNAME=${{MySQL.MYSQL_USER}}
SPRING_DATASOURCE_PASSWORD=${{MySQL.MYSQL_PASSWORD}}
SERVER_PORT=8080
COMPILER_GCC_PATH=gcc
```

5. 点击 "Update Variables"
6. 服务会自动重新部署

### 2.4 生成域名

1. 点击 "Settings" 标签
2. 找到 "Domains" 部分
3. 点击 "Generate Domain"
4. 复制生成的域名（例如：`your-app.up.railway.app`）

### 2.5 初始化数据库

1. 点击 MySQL 服务
2. 点击 "Data" 标签
3. 点击 "Query" 按钮
4. 打开项目中的 `springboot/src/main/resources/sql/init.sql`
5. 复制所有内容，粘贴到 Query 窗口
6. 点击 "Run" 执行

### 2.6 测试后端

在浏览器访问：`https://your-app.up.railway.app/api/health`

如果看到 `{"status":"UP",...}`，说明后端部署成功！✅

## 第 3 步：部署前端到 Vercel

### 3.1 导入项目

1. 访问 https://vercel.com/
2. 点击 "Sign Up" → "Continue with GitHub"
3. 授权 Vercel 访问你的 GitHub
4. 点击 "Add New..." → "Project"
5. 找到你的 GitHub 仓库，点击 "Import"

### 3.2 配置项目

在配置页面填写：

- **Framework Preset**: Vite
- **Root Directory**: 点击 "Edit"，输入 `vue`
- **Build Command**: `npm run build`
- **Output Directory**: `dist`
- **Install Command**: `npm install`

### 3.3 配置环境变量

在 "Environment Variables" 部分：

- **Name**: `VITE_API_BASE_URL`
- **Value**: `https://your-app.up.railway.app`（你的 Railway 后端地址）

⚠️ 注意：不要在 URL 末尾加 `/api`

### 3.4 部署

1. 点击 "Deploy"
2. 等待构建完成（约 2-3 分钟）
3. 部署成功后，Vercel 会显示你的网站 URL

### 3.5 访问你的应用

点击 Vercel 提供的 URL（例如：`your-app.vercel.app`）

## 🐛 遇到问题？

### 后端无法访问

1. 检查 Railway 部署日志
2. 确认环境变量配置正确
3. 确认数据库已初始化

### 前端无法连接后端

1. 检查 Vercel 的 `VITE_API_BASE_URL` 环境变量
2. 确认后端 URL 正确（不要有多余的斜杠）
3. 打开浏览器开发者工具（F12）查看网络请求

### 数据库连接失败

1. 确认 MySQL 服务正在运行
2. 检查环境变量引用语法
3. 重新部署后端服务
