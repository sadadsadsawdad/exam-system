# 🚀 立即推送代码到 GitHub

## ❌ 当前问题

推送失败：`Connection was reset`（连接被重置）

这是网络问题，通常由以下原因引起：
- GitHub 服务器连接不稳定
- 防火墙或代理设置
- 文件太大

---

## ✅ 解决方案（按推荐顺序）

### 方案 1：使用 GitHub Desktop（最简单，强烈推荐）

1. **下载 GitHub Desktop**
   - 访问：https://desktop.github.com/
   - 下载并安装

2. **打开项目**
   - 打开 GitHub Desktop
   - File → Add Local Repository
   - 选择项目文件夹：`D:\Users\l\Desktop\exam`

3. **推送代码**
   - 点击右上角 "Publish repository" 或 "Push origin"
   - 完成！

**优点**：图形界面，自动处理网络问题，最稳定

---

### 方案 2：配置 Git 使用 IPv4（快速有效）

在终端执行：

```bash
git config --global http.version HTTP/1.1
git config --global http.postBuffer 524288000
git config --global core.compression 0

# 然后重试推送
git push -u origin main
```

---

### 方案 3：使用 SSH 方式（如果配置了 SSH key）

```bash
# 切换到 SSH 方式
git remote set-url origin git@github.com:sadadsadsawdad/exam-system.git

# 推送
git push -u origin main
```

如果没有配置 SSH key，参考：https://docs.github.com/zh/authentication/connecting-to-github-with-ssh

---

### 方案 4：分批推送（如果文件太大）

```bash
# 先推送部分文件
git push -u origin main --no-verify

# 如果还是失败，尝试浅克隆
git push -u origin main --depth=1
```

---

### 方案 5：使用代理（如果你有 VPN）

```bash
# 如果使用代理，配置代理端口（常见端口：7890, 1080, 10809）
git config --global http.proxy http://127.0.0.1:7890
git config --global https.proxy http://127.0.0.1:7890

# 推送
git push -u origin main

# 推送成功后，取消代理设置
git config --global --unset http.proxy
git config --global --unset https.proxy
```

---

### 方案 6：直接在 GitHub 网页上传（备用方案）

1. 访问：https://github.com/sadadsadsawdad/exam-system
2. 点击 "Add file" → "Upload files"
3. 将项目文件夹拖拽到网页
4. 点击 "Commit changes"

**缺点**：需要手动上传，但保证能成功

## 🆘 还是不行？

如果所有方案都失败，可以：

1. **使用 Gitee（国内 GitHub）**
   - 访问：https://gitee.com/
   - 创建仓库并推送
   - Railway 和 Vercel 也支持 Gitee

2. **稍后重试**
   - GitHub 服务器可能暂时不稳定
   - 等待 10-30 分钟后重试

3. **检查网络**
   - 确认能访问 github.com
   - 尝试切换网络（WiFi/手机热点）
