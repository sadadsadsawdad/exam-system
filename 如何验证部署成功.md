# ✅ 如何验证部署成功

## 📊 Railway 后端部署验证

### 1. 查看部署状态

在 Railway 项目页面：

**成功的标志**：
- ✅ 服务卡片显示绿色的 "Active" 状态
- ✅ 右上角显示绿色的勾号 ✓
- ✅ 没有红色的错误提示

**失败的标志**：
- ❌ 显示红色的 "Failed" 或 "Crashed"
- ❌ 有错误信息提示
- ❌ 服务一直在重启

### 2. 查看构建日志

点击你的后端服务 → 点击 "Deployments" 标签：

**成功的日志应该包含**：
```
✓ Building...
✓ Maven build successful
✓ Starting application
✓ Application started on port 8080
```

**如果失败，会看到**：
```
✗ Build failed
✗ Error: ...
```

### 3. 测试后端 API

#### 方法 1：浏览器测试

在浏览器地址栏输入（替换为你的实际域名）：
```
https://your-app.up.railway.app/api/health
```

**成功的响应**：
```json
{
  "status": "UP",
  "message": "Application is running",
  "timestamp": 1234567890
}
```

**失败的响应**：
- 404 Not Found
- 502 Bad Gateway
- 连接超时

#### 方法 2：使用 curl 测试

在终端执行：
```bash
curl https://your-app.up.railway.app/api/health
```

### 4. 检查数据库连接

在 Railway 项目页面：

- 点击 MySQL 服务
- 查看状态是否为 "Active"
- 点击 "Data" 标签
- 确认能看到数据库表（users, questions, exams 等）

### 5. 查看环境变量

点击后端服务 → "Variables" 标签：

**必须配置的变量**：
- ✅ SPRING_DATASOURCE_URL
- ✅ SPRING_DATASOURCE_USERNAME
- ✅ SPRING_DATASOURCE_PASSWORD
- ✅ SERVER_PORT
- ✅ COMPILER_GCC_PATH

## 📊 Vercel 前端部署验证

### 1. 查看部署状态

在 Vercel 项目页面：

**成功的标志**：
- ✅ 显示 "Ready" 状态
- ✅ 有一个可点击的 URL
- ✅ 构建时间显示（例如：2m 30s）

**失败的标志**：
- ❌ 显示 "Failed" 或 "Error"
- ❌ 构建日志中有错误信息

### 2. 查看构建日志

点击部署记录 → 查看 "Building" 日志：

**成功的日志应该包含**：
```
✓ Installing dependencies
✓ Building application
✓ Build completed
✓ Deployment ready
```

**如果失败，会看到**：
```
✗ Build failed
✗ Error: ...
```

### 3. 访问前端网站

点击 Vercel 提供的 URL（例如：`your-app.vercel.app`）

**成功的标志**：
- ✅ 看到登录页面
- ✅ 页面样式正常显示
- ✅ 没有空白页面

**失败的标志**：
- ❌ 页面空白
- ❌ 显示 404 错误
- ❌ 样式错乱

### 4. 测试前后端连接

打开浏览器开发者工具（按 F12）：

1. 切换到 "Network" (网络) 标签
2. 刷新页面
3. 查看 API 请求

**成功的标志**：
- ✅ 看到对后端的请求（例如：`/api/users`）
- ✅ 请求状态码为 200
- ✅ 能看到返回的数据

**失败的标志**：
- ❌ 请求状态码为 404、500、502
- ❌ 请求失败（红色）
- ❌ CORS 错误

### 5. 测试登录功能

使用默认账号登录：
- 用户名：`admin`
- 密码：`admin123`

**成功的标志**：
- ✅ 能成功登录
- ✅ 跳转到主页
- ✅ 能看到用户信息

**失败的标志**：
- ❌ 登录失败
- ❌ 提示网络错误
- ❌ 页面无响应

## 🐛 常见问题和解决方案

### Railway 后端问题

#### 问题 1：服务一直重启

**原因**：数据库连接失败或环境变量配置错误

**解决**：
1. 检查环境变量是否正确
2. 确认 MySQL 服务正在运行
3. 查看部署日志找到具体错误

#### 问题 2：构建失败

**原因**：Maven 构建错误或依赖下载失败

**解决**：
1. 查看构建日志
2. 检查 `pom.xml` 是否正确
3. 等待几分钟后重新部署

#### 问题 3：访问 /api/health 返回 404

**原因**：应用未正确启动或路由配置错误

**解决**：
1. 确认服务状态为 "Active"
2. 检查域名是否正确
3. 查看应用日志

### Vercel 前端问题

#### 问题 1：页面空白

**原因**：构建配置错误或路径不正确

**解决**：
1. 确认 Root Directory 设置为 `vue`
2. 确认 Build Command 为 `npm run build`
3. 确认 Output Directory 为 `dist`

#### 问题 2：API 请求失败

**原因**：`VITE_API_BASE_URL` 配置错误

**解决**：
1. 检查环境变量是否正确
2. 确认后端 URL 没有多余的斜杠
3. 确认后端服务正在运行

#### 问题 3：CORS 错误

**原因**：后端 CORS 配置问题

**解决**：
1. 确认后端已部署 `WebConfig.java`
2. 重新部署后端
3. 清除浏览器缓存
