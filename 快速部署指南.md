# 快速部署指南

这是一个简化的部署指南，帮助你快速将项目部署到服务器。

## 前提条件

- 一台 Linux 服务器（Ubuntu 20.04+ 或 CentOS 7+）
- 服务器至少 2GB 内存
- 有 sudo 权限
- 服务器可以访问互联网

## 部署步骤

### 第一步：上传项目到服务器

```bash
# 在本地打包项目
tar -czf exam-system.tar.gz springboot/ vue/ deploy/ 部署文档.md

# 上传到服务器（替换为你的服务器 IP）
scp exam-system.tar.gz user@your-server-ip:/home/user/

# 登录服务器
ssh user@your-server-ip

# 解压项目
cd /home/user
tar -xzf exam-system.tar.gz
cd exam-system
```

### 第二步：初始化服务器环境

```bash
# 运行环境配置脚本（需要 sudo 权限）
sudo bash deploy/setup-server.sh
```

这个脚本会自动安装：
- JDK 17
- MySQL 8.0
- Nginx
- Maven
- Node.js 20
- GCC

### 第三步：配置数据库

```bash
# 登录 MySQL
sudo mysql -u root -p

# 执行以下 SQL（修改密码）
CREATE DATABASE exam CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER 'examuser'@'localhost' IDENTIFIED BY '你的强密码';
GRANT ALL PRIVILEGES ON exam.* TO 'examuser'@'localhost';
FLUSH PRIVILEGES;
EXIT;

# 导入初始化数据
mysql -u examuser -p exam < springboot/src/main/resources/sql/init.sql
```

### 第四步：修改配置文件

#### 1. 修改后端配置

编辑 `springboot/src/main/resources/application.properties`：

```properties
spring.datasource.password=你的数据库密码
```

#### 2. 修改 Nginx 配置

编辑 `deploy/nginx.conf`：

```nginx
server_name your-server-ip;  # 改为你的服务器 IP 或域名
```

### 第五步：安装服务配置

```bash
# 安装 systemd 服务
sudo cp deploy/exam-backend.service /etc/systemd/system/
sudo systemctl daemon-reload
sudo systemctl enable exam-backend

# 安装 Nginx 配置
sudo cp deploy/nginx.conf /etc/nginx/sites-available/exam-system
sudo ln -s /etc/nginx/sites-available/exam-system /etc/nginx/sites-enabled/
sudo nginx -t
```

### 第六步：一键部署

```bash
# 给脚本添加执行权限
chmod +x deploy/*.sh

# 执行一键部署
bash deploy/quick-deploy.sh
```

### 第七步：验证部署

```bash
# 检查后端服务
sudo systemctl status exam-backend

# 检查 Nginx
sudo systemctl status nginx

# 访问测试
curl http://localhost
```

在浏览器访问：`http://your-server-ip`

## 常见问题

### 1. 后端服务启动失败

```bash
# 查看日志
sudo journalctl -u exam-backend -n 50

# 常见原因：
# - 数据库连接失败：检查密码和数据库是否创建
# - 端口被占用：检查 8081 端口
# - 权限问题：确保 /opt/exam-backend 目录权限正确
```

### 2. 前端页面无法访问

```bash
# 检查 Nginx 错误日志
sudo tail -f /var/log/nginx/error.log

# 测试 Nginx 配置
sudo nginx -t

# 常见原因：
# - 配置文件语法错误
# - 端口被占用
# - 文件权限问题
```

### 3. API 请求失败

```bash
# 检查后端是否运行
sudo systemctl status exam-backend

# 检查端口监听
sudo netstat -tlnp | grep 8081

# 测试 API
curl http://localhost:8081/api/
```

## 更新部署

当代码有更新时：

```bash
# 上传新代码到服务器
# 然后执行：
bash deploy/quick-deploy.sh
```

## 配置 HTTPS（可选）

```bash
# 安装 Certbot
sudo apt install certbot python3-certbot-nginx -y

# 获取证书（替换为你的域名）
sudo certbot --nginx -d your-domain.com

# 证书会自动续期
```

## 配置数据库备份

```bash
# 修改备份脚本中的密码
nano deploy/backup-db.sh

# 添加执行权限
chmod +x deploy/backup-db.sh

# 设置定时任务（每天凌晨 2 点备份）
sudo crontab -e
# 添加：
0 2 * * * /home/user/exam-system/deploy/backup-db.sh >> /var/log/db-backup.log 2>&1
```

## 监控和维护

```bash
# 查看服务状态
sudo systemctl status exam-backend nginx mysql

# 查看日志
sudo journalctl -u exam-backend -f
sudo tail -f /var/log/nginx/exam-access.log

# 查看系统资源
htop
df -h
free -h
```
