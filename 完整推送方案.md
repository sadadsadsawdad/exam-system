## 🎯 方案 A：使用 GitHub Desktop（最推荐）

### 为什么推荐？
- 官方工具，自动处理网络问题
- 图形界面，简单易用
- 成功率最高

### 操作步骤

1. **下载安装**
   - 访问：https://desktop.github.com/
   - 下载并安装（约 100MB，5分钟）

2. **登录 GitHub**
   - 打开 GitHub Desktop
   - File → Options → Accounts
   - 点击 "Sign in" 登录你的 GitHub 账号

3. **添加项目**
   - File → Add local repository
   - 点击 "Choose..." 按钮
   - 选择：`D:\Users\l\Desktop\exam`
   - 点击 "Add repository"

4. **推送代码**
   - 在主界面会看到你的更改
   - 点击右上角 "Push origin" 按钮
   - 等待完成（可能需要几分钟）

5. **验证**
   - 访问：https://github.com/sadadsadsawdad/exam-system
   - 确认代码已上传

## 🎯 方案 B：使用 Gitee（国内替代，速度快）

如果 GitHub 一直连接不上，可以使用 Gitee：

### 1. 创建 Gitee 仓库

1. 访问：https://gitee.com/
2. 注册/登录账号
3. 点击右上角 "+" → "新建仓库"
4. 填写：
   - 仓库名称：`exam-system`
   - 路径：`exam-system`
   - 可见性：公开或私有
5. 点击 "创建"

### 2. 推送到 Gitee

在终端执行：

```bash
# 添加 Gitee 远程仓库
git remote add gitee https://gitee.com/你的用户名/exam-system.git

# 推送到 Gitee
git push -u gitee main
```

### 3. 同步到 GitHub

Gitee 可以自动同步到 GitHub：

1. 在 Gitee 仓库页面
2. 点击 "管理" → "仓库镜像管理"
3. 添加 GitHub 仓库地址
4. 设置自动同步

## 🎯 方案 C：使用压缩包上传

如果所有方式都失败，直接上传：

### 1. 打包项目

在项目目录执行：

```bash
# 创建压缩包（排除 .git 文件夹）
tar -czf exam-system.tar.gz --exclude=.git --exclude=node_modules --exclude=target .
```

或者手动：
1. 复制整个项目文件夹
2. 删除 `.git`、`node_modules`、`target` 文件夹
3. 压缩为 zip 文件

### 2. 上传到 GitHub

1. 访问：https://github.com/sadadsadsawdad/exam-system
2. 点击 "Add file" → "Upload files"
3. 拖拽文件或选择文件
4. 点击 "Commit changes"

## 🎯 方案 D：修复网络问题后重试

### 检查网络

```bash
# 测试 GitHub 连接
ping github.com

# 测试 HTTPS 连接
curl -I https://github.com
```

### 配置代理（如果使用 VPN）

```bash
# 查看代理端口（常见：7890, 1080, 10809）
# 配置 Git 代理
git config --global http.proxy http://127.0.0.1:7890
git config --global https.proxy http://127.0.0.1:7890

# 推送
git push -u origin main

# 成功后取消代理
git config --global --unset http.proxy
git config --global --unset https.proxy
```

### 使用 SSH 方式

```bash
# 切换到 SSH
git remote set-url origin git@github.com:sadadsadsawdad/exam-system.git

# 推送
git push -u origin main
```
