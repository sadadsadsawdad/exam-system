# Railway + Vercel 部署检查清单

### 1. 代码准备
- [ ] 所有代码已提交到 Git

### 2. 配置文件检查
- [ ] `railway.json` 存在于项目根目录
- [ ] `vercel.json` 存在于项目根目录

- [ ] `springboot/src/main/resources/sql/init.sql` 文件存在
- [ ] SQL 文件包含所有必要的表结构和初始数据

## 🚂 Railway 部署步骤

### 第一步：创建项目
- [ ] 访问 https://railway.app/
- [ ] 使用 GitHub 登录
- [ ] 创建新项目并连接 GitHub 仓库

### 第二步：等待自动构建
- [ ] Railway 自动检测到 `railway.json`
- [ ] 构建成功（查看日志确认）
- [ ] 服务状态显示为 "Active"

### 第三步：添加 MySQL
- [ ] 点击 "+ New" → "Database" → "Add MySQL"
- [ ] MySQL 创建成功
- [ ] 复制 MySQL 连接信息

### 第四步：配置环境变量
在后端服务的 Variables 中添加：

```bash
SPRING_DATASOURCE_URL=jdbc:mysql://${{MySQL.MYSQL_HOST}}:${{MySQL.MYSQL_PORT}}/${{MySQL.MYSQL_DATABASE}}?useSSL=false&serverTimezone=Asia/Shanghai&characterEncoding=utf8&allowPublicKeyRetrieval=true
SPRING_DATASOURCE_USERNAME=${{MySQL.MYSQL_USER}}
SPRING_DATASOURCE_PASSWORD=${{MySQL.MYSQL_PASSWORD}}
SERVER_PORT=8080
COMPILER_GCC_PATH=gcc
```

- [ ] 环境变量已添加
- [ ] 点击 "Deploy" 重新部署
- [ ] 部署成功

### 第五步：获取后端 URL
- [ ] 在 Settings → Domains 中生成域名
- [ ] 复制域名（例如：`your-app.up.railway.app`）
- [ ] 测试访问：`https://your-app.up.railway.app/api/health`

### 第六步：初始化数据库
- [ ] 连接到 Railway MySQL
- [ ] 导入 `init.sql` 文件
- [ ] 验证表和数据已创建

---

## ▲ Vercel 部署步骤

### 第一步：导入项目
- [ ] 访问 https://vercel.com/
- [ ] 使用 GitHub 登录
- [ ] 点击 "Add New..." → "Project"
- [ ] 选择你的 GitHub 仓库

### 第二步：配置项目
- [ ] Framework Preset: Vite
- [ ] Root Directory: `vue`
- [ ] Build Command: `npm run build`
- [ ] Output Directory: `dist`
- [ ] Install Command: `npm install`

### 第三步：配置环境变量
- [ ] 添加环境变量 `VITE_API_BASE_URL`
- [ ] 值设置为 Railway 后端 URL（例如：`https://your-app.up.railway.app`）
- [ ] 注意：不要在 URL 末尾加 `/api`

### 第四步：部署
- [ ] 点击 "Deploy"
- [ ] 等待构建完成（2-3分钟）
- [ ] 部署成功

### 第五步：获取前端 URL
- [ ] 复制 Vercel 提供的 URL（例如：`your-app.vercel.app`）
- [ ] 访问 URL 查看应用

## 🐛 故障排查
