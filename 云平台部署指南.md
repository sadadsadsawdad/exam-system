# 云平台部署指南

本指南介绍如何将在线考试系统部署到各种云平台。

## 部署架构

由于这是一个前后端分离项目：
- **前端（Vue）**：可部署到 Vercel、Netlify、GitHub Pages 等静态托管平台
- **后端（Spring Boot）**：需要部署到支持 Java 的云平台（如 Railway、Render、Heroku、阿里云、腾讯云等）
- **数据库（MySQL）**：使用云数据库服务

---

## 方案一：Railway（推荐 - 最简单）

Railway 支持 Java 应用和 MySQL，适合快速部署。

### 1. 部署后端到 Railway

#### 准备工作

创建 `railway.json`（已为你创建）：

```json
{
  "$schema": "https://railway.app/railway.schema.json",
  "build": {
    "builder": "NIXPACKS",
    "buildCommand": "cd springboot && mvn clean package -DskipTests"
  },
  "deploy": {
    "startCommand": "java -jar springboot/target/springboot-0.0.1-SNAPSHOT.jar",
    "restartPolicyType": "ON_FAILURE",
    "restartPolicyMaxRetries": 10
  }
}
```

#### 部署步骤

1. 访问 [Railway.app](https://railway.app/)
2. 使用 GitHub 账号登录
3. 点击 "New Project" → "Deploy from GitHub repo"
4. 选择你的项目仓库
5. Railway 会自动检测 Spring Boot 项目并部署

#### 配置环境变量

在 Railway 项目设置中添加：

```
SPRING_DATASOURCE_URL=jdbc:mysql://[数据库地址]:3306/exam
SPRING_DATASOURCE_USERNAME=examuser
SPRING_DATASOURCE_PASSWORD=你的密码
SERVER_PORT=8080
```

#### 添加 MySQL 数据库

1. 在 Railway 项目中点击 "New" → "Database" → "Add MySQL"
2. 复制数据库连接信息到环境变量
3. 使用 Railway 提供的数据库管理工具导入 `init.sql`

### 2. 部署前端到 Vercel

1. 访问 [Vercel.com](https://vercel.com/)
2. 导入 GitHub 仓库
3. 配置构建设置：
   - Framework Preset: Vite
   - Root Directory: `vue`
   - Build Command: `npm run build`
   - Output Directory: `dist`
4. 添加环境变量：
   ```
   VITE_API_BASE_URL=https://your-backend.railway.app
   ```
5. 部署

## 方案二：Render

### 1. 部署后端

1. 访问 [Render.com](https://render.com/)
2. 创建新的 Web Service
3. 连接 GitHub 仓库
4. 配置：
   - Environment: Java
   - Build Command: `cd springboot && mvn clean package -DskipTests`
   - Start Command: `java -jar springboot/target/springboot-0.0.1-SNAPSHOT.jar`
5. 添加环境变量（同 Railway）

### 2. 添加 MySQL 数据库

Render 不直接提供 MySQL，可以使用：
- PlanetScale（免费 MySQL）
- Railway MySQL
- 阿里云 RDS

### 3. 部署前端

同 Vercel 方案

## 方案三：阿里云/腾讯云（国内推荐）

### 1. 购买服务器

- 阿里云 ECS 或腾讯云 CVM
- 配置：2核2G 起步
- 系统：Ubuntu 20.04

### 2. 使用自动化脚本部署

参考项目根目录的 `部署文档.md` 和 `deploy/` 目录下的脚本。

```bash
# 上传项目到服务器
scp -r . user@your-server:/home/user/exam-system

# SSH 登录服务器
ssh user@your-server

# 执行部署
cd exam-system
sudo bash deploy/setup-server.sh
bash deploy/quick-deploy.sh
```

### 3. 配置域名

1. 在域名服务商添加 A 记录指向服务器 IP
2. 配置 Nginx（已包含在部署脚本中）
3. 使用 Let's Encrypt 配置 HTTPS

## 方案四：Docker 部署（适合任何云平台）

### 1. 使用提供的 Docker 配置

项目已包含 `docker-compose.yml`，一键启动所有服务：

```bash
# 构建并启动
docker-compose up -d

# 查看日志
docker-compose logs -f

# 停止服务
docker-compose down
```

### 2. 部署到云平台

支持 Docker 的云平台：
- AWS ECS
- Google Cloud Run
- Azure Container Instances
- 阿里云容器服务
- 腾讯云容器服务

## 方案五：Serverless 部署

### 后端改造为 Serverless

Spring Boot 可以通过 AWS Lambda 或阿里云函数计算部署，但需要一些改造：

1. 添加 AWS Lambda 适配器
2. 修改启动类
3. 配置 API Gateway

（这个方案较复杂，不推荐初学者使用）

## 快速开始：Railway + Vercel（5分钟部署）

### 第一步：准备代码

```bash
# 1. 提交代码到 GitHub
git add .
git commit -m "准备部署"
git push origin main
```

### 第二步：部署后端（Railway）

1. 访问 https://railway.app/
2. 点击 "Start a New Project"
3. 选择 "Deploy from GitHub repo"
4. 选择你的仓库
5. 添加 MySQL 数据库
6. 配置环境变量
7. 等待部署完成，复制后端 URL

### 第三步：部署前端（Vercel）

1. 访问 https://vercel.com/
2. 点击 "Import Project"
3. 选择你的 GitHub 仓库
4. 配置：
   - Root Directory: `vue`
   - Framework: Vite
5. 添加环境变量：
   ```
   VITE_API_BASE_URL=https://your-backend.railway.app
   ```
6. 点击 "Deploy"

### 第四步：初始化数据库

1. 在 Railway 中打开 MySQL 数据库
2. 使用提供的连接信息连接数据库
3. 导入 `springboot/src/main/resources/sql/init.sql`

完成！访问 Vercel 提供的 URL 即可使用。

## 注意事项

### 1. API 地址配置

确保前端正确配置了后端 API 地址：

- 开发环境：`http://localhost:8081`
- 生产环境：你的后端部署地址

### 2. CORS 配置

后端需要配置 CORS 允许前端域名访问（已在代码中配置）。

### 3. 数据库连接

- 使用云数据库时，注意配置白名单
- 生产环境使用强密码
- 定期备份数据

### 4. 环境变量

敏感信息（如数据库密码）使用环境变量，不要提交到代码仓库。

### 5. 性能优化

- 前端启用 CDN
- 后端配置缓存
- 数据库添加索引
- 使用连接池

## 故障排查

### 前端无法连接后端

1. 检查 `VITE_API_BASE_URL` 环境变量
2. 检查后端 CORS 配置
3. 查看浏览器控制台网络请求

### 后端启动失败

1. 查看部署平台日志
2. 检查数据库连接配置
3. 确认 Java 版本（需要 JDK 17）

### 数据库连接失败

1. 检查数据库地址和端口
2. 确认用户名密码正确
3. 检查数据库白名单配置
