# 一键部署到云平台

## 🚀 最简单的方式：使用 Docker Compose

### 前提条件
- 安装 Docker 和 Docker Compose
- 有一台服务器或本地环境

### 一键启动

```bash
# 1. 克隆或上传项目到服务器

# 2. 启动所有服务（数据库 + 后端 + 前端）
docker-compose up -d

# 3. 查看日志
docker-compose logs -f

# 4. 访问应用
# 浏览器打开: http://your-server-ip
```

就这么简单！Docker Compose 会自动：
- 启动 MySQL 数据库并初始化数据
- 构建并启动 Spring Boot 后端
- 构建并启动 Vue 前端 + Nginx

### 管理命令

```bash
# 停止所有服务
docker-compose down

# 重启服务
docker-compose restart

# 查看服务状态
docker-compose ps

# 查看特定服务日志
docker-compose logs backend
docker-compose logs frontend
docker-compose logs mysql
```

## 🌐 部署到 Railway（免费额度）

### 步骤 1：准备 GitHub 仓库

```bash
# 初始化 Git（如果还没有）
git init
git add .
git commit -m "准备部署到 Railway"

# 推送到 GitHub
git remote add origin https://github.com/你的用户名/你的仓库.git
git push -u origin main
```

### 步骤 2：部署后端

1. 访问 https://railway.app/
2. 点击 "Start a New Project"
3. 选择 "Deploy from GitHub repo"
4. 选择你的仓库
5. Railway 会自动检测 `railway.json` 并开始构建

### 步骤 3：添加 MySQL 数据库

1. 在 Railway 项目中点击 "New" → "Database" → "Add MySQL"
2. 等待数据库创建完成
3. 点击 MySQL 服务，复制连接信息

### 步骤 4：配置环境变量

在后端服务的 Variables 中添加：

```
SPRING_DATASOURCE_URL=jdbc:mysql://[MySQL_HOST]:[MySQL_PORT]/railway
SPRING_DATASOURCE_USERNAME=[MySQL_USER]
SPRING_DATASOURCE_PASSWORD=[MySQL_PASSWORD]
SERVER_PORT=8080
```

（Railway 会自动提供这些变量，你也可以直接使用）

### 步骤 5：初始化数据库

1. 在 Railway 中打开 MySQL 数据库
2. 点击 "Connect" 获取连接命令
3. 使用 MySQL 客户端连接并导入 `init.sql`：

```bash
mysql -h [host] -P [port] -u [user] -p[password] railway < springboot/src/main/resources/sql/init.sql
```

### 步骤 6：部署前端到 Vercel

1. 访问 https://vercel.com/
2. 点击 "Import Project"
3. 选择你的 GitHub 仓库
4. 配置：
   - Root Directory: `vue`
   - Framework Preset: Vite
   - Build Command: `npm run build`
   - Output Directory: `dist`
5. 添加环境变量：
   ```
   VITE_API_BASE_URL=https://你的后端地址.railway.app
   ```
6. 点击 "Deploy"

完成！访问 Vercel 提供的 URL。

## 🐳 部署到任何支持 Docker 的云平台

### 阿里云/腾讯云容器服务

1. 构建镜像：
```bash
docker build -t exam-system:latest .
```

2. 推送到容器镜像仓库：
```bash
# 登录阿里云容器镜像服务
docker login --username=你的用户名 registry.cn-hangzhou.aliyuncs.com

# 打标签
docker tag exam-system:latest registry.cn-hangzhou.aliyuncs.com/你的命名空间/exam-system:latest

# 推送
docker push registry.cn-hangzhou.aliyuncs.com/你的命名空间/exam-system:latest
```

## 📱 部署到 Render

### 部署后端

1. 访问 https://render.com/
2. 点击 "New +" → "Web Service"
3. 连接 GitHub 仓库
4. 配置：
   - Name: exam-backend
   - Environment: Docker
   - Dockerfile Path: `springboot/Dockerfile`
   - Instance Type: Free
5. 添加环境变量（数据库配置）
6. 点击 "Create Web Service"

### 添加数据库

Render 不提供 MySQL，可以：
- 使用 PlanetScale（免费 MySQL）
- 使用 Railway MySQL
- 使用其他云数据库

### 部署前端

同 Vercel 方案

## 🎯 推荐方案

### 个人学习/演示项目
**Railway + Vercel**
- 优点：免费额度充足，部署简单
- 缺点：国外服务器，国内访问可能较慢

### 国内访问
**阿里云/腾讯云 + Docker Compose**
- 优点：国内访问快，稳定
- 缺点：需要购买服务器（学生机很便宜）

### 快速测试
**本地 Docker Compose**
- 优点：最快，一条命令搞定
- 缺点：只能本地访问

## 🔧 故障排查

### Docker Compose 启动失败

```bash
# 查看详细日志
docker-compose logs

# 重新构建
docker-compose build --no-cache
docker-compose up -d
```

### 后端连接数据库失败

检查环境变量配置是否正确：
```bash
docker-compose exec backend env | grep SPRING_DATASOURCE
```

### 前端无法访问后端 API

1. 检查 `VITE_API_BASE_URL` 环境变量
2. 检查后端 CORS 配置
3. 查看浏览器控制台网络请求
